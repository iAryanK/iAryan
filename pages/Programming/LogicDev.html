Logic Development in Programming

August 3, 2022 by Aryan

Let us see the continuous set of process in which a software is built and managed. The steps involved are program analysis, program design, program development, documentation and maintainance. These steps are further divided into sub-steps. (Refer to the table of content.)

Program Analysis

Understand the problem
Let's see an example.
Suppose we have to develop an application for a bank. The first step in program analysis is that we have to understand the problem. We need to dive deep into almost every aspect of how a software for a bank can be designed; what all features it must contain etc. using a lot of research from all possible sources.
Find multiple solutions
Now for each and every sub-problem, we need to think of multiple solutions. As in our application for bank, we need to figure out how each and every component of the app should be designed. What can be the various possible processes for its fast and smooth user experience.
Select a solution
From solutions that are available to us, choose the optimal solution to be implemented. Optimal solution is the best solution in our limited avaiable resources.

Program Design

Write Algorithm
After we have chosen the optimal solution for each and every part of the problem, we need to think of how to implement it in our program. The first step in program design is to write the algorithm i.e. the correct sequence and procedure of instructions required to carry out the task.
Draw Flowchart
Next, we draw a flowchart with its diagrammatic representation that illustrates the sequence of operations to be performed to arrive at the solution.
Write Pseudocode
Finally, we write the pseudocode i.e. design tool developed within structured programming.

Program Development

Convert pseudocode to executable code
After we have designed our program of how the features and design of a bank application is created, we now convert our flowchart and pseudocode into actual code that will be executable.
For this, we choose a programming language. There may be multiple programming languages which will be advantageous to be used for designing a bank software, but we will try to work in the one which we know the best.
Compile and Execute
Now, we have converted our pseudocode into executable one. We will compile the program and convert it into a binary executable file and check, if any, syntactical error is there in our source code.
Test and Install
The compiled program is executed. Next, the program is tested on various sample data avaiable. This is to check if our program works well in real world as in simulator.
The program is then installed on devices and can be used by the user.

Program documentation and maintainance

This is the last but the most important part of a software development. The programmer needs to write documentation on how he managed to design the software and how to source code is written to create every single functionality.
The documentation is written in layman terms so that others may understand it easily.

The program also requires maintainance. It includes updating the software- where bugs are fixed and new features are added. As in our example of developing a software application for a bank, the software requires regular updating. Also, the code is optimized for fast execution of the program.
